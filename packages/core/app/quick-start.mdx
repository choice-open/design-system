# Quick Start Guide

Get up and running with @choice-ui/react in minutes. This guide covers installation, configuration, and key concepts to help you build beautiful interfaces efficiently.

## Installation

Install the design system and design tokens using your preferred package manager:

```bash
# Using pnpm (recommended)
pnpm add @choice-ui/react @choice-ui/design-tokens

# Using npm
npm install @choice-ui/react @choice-ui/design-tokens

# Using yarn
yarn add @choice-ui/react @choice-ui/design-tokens
```

## CSS Configuration

### Import Required Styles

Add these imports to your main CSS file (e.g., `app.css` or `index.css`):

```css
@import "tailwindcss";
@import "@choice-ui/design-tokens/tokens.css";
@import "@choice-ui/design-tokens/preflight.css";
@import "@choice-ui/design-tokens/tailwind.css";
@source "../../../node_modules/@choice-ui/react";
```

**Note:** The old `@import "@choice-ui/react/styles/theme.css"` is no longer needed and should be removed.

Learn more about design tokens at [https://choice-ui.com/tokens](https://choice-ui.com/tokens)

### Understanding the @source Directive

The `@source` directive is a powerful Tailwind CSS 4.0 feature that tells Tailwind where to scan for class names:

#### What @source Does

- **Content Detection**: Instructs Tailwind to scan specific directories for class usage
- **Automatic Scanning**: Includes component library files in the build process
- **Performance Optimization**: Ensures only necessary styles are generated
- **Module Integration**: Seamlessly integrates third-party component libraries

#### Why You Need It

```css
@source "../../../node_modules/@choice-ui/react";
```

This directive is essential because:

1. **Library Integration**: Ensures Tailwind finds classes used in the design system components
2. **Build Optimization**: Prevents unused component styles from being purged
3. **Automatic Discovery**: Eliminates manual configuration of content paths
4. **Cross-Package Support**: Enables style sharing across monorepo packages

## Assemble a Component Philosophy

Our design system embraces the **"Assemble a Component"** approach, a modern methodology for building scalable and maintainable user interfaces.

### Core Concept

Instead of monolithic components, we provide **composable building blocks** that can be assembled to create complex interfaces:

```tsx
// × Monolithic approach
<DataTable
  data={users}
  columns={columns}
  pagination={true}
  sorting={true}
  filtering={true}
/>

// ○ Assemble a Component approach
<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head sortable>Name</Table.Head>
      <Table.Head>Email</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    {users.map(user => (
      <Table.Row key={user.id}>
        <Table.Cell>{user.name}</Table.Cell>
        <Table.Cell>{user.email}</Table.Cell>
      </Table.Row>
    ))}
  </Table.Body>
</Table>
```

### Key Advantages

#### **Flexibility**

- Mix and match components as needed
- Easy customization without prop drilling
- Adaptable to unique design requirements

#### **Composability**

- Small, focused components with single responsibilities
- Natural component composition patterns
- Reusable across different contexts

#### **Scalability**

- Easy to extend with new variants
- Maintainable component architecture
- Future-proof design patterns

#### **Design Consistency**

- Shared design tokens across all components
- Consistent spacing, typography, and colors
- Unified interaction patterns

## Theme Configuration

Configure theme mode by adding the appropriate class or attribute to your HTML element:

### Option 1: CSS Class (Recommended)

```html
<!-- Light mode -->
<html class="light">
  <!-- Your app content -->
</html>

<!-- Dark mode -->
<html class="dark">
  <!-- Your app content -->
</html>
```

### Option 2: Data Attribute

```html
<!-- Light mode -->
<html data-theme="light">
  <!-- Your app content -->
</html>

<!-- Dark mode -->
<html data-theme="dark">
  <!-- Your app content -->
</html>
```

### Dynamic Theme Switching

```tsx
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState("light")

  useEffect(() => {
    document.documentElement.className = theme
    // or
    document.documentElement.setAttribute("data-theme", theme)
  }, [theme])

  return <ThemeContext.Provider value={{ theme, setTheme }}>{children}</ThemeContext.Provider>
}
```

## Basic Usage

Once configured, you can start using components immediately:

```tsx
import { Button, Input, Card } from "@choice-ui/react"

export default function App() {
  return (
    <div className="p-4">
      <Card>
        <Card.Header>
          <Card.Title>Welcome to Design System</Card.Title>
        </Card.Header>
        <Card.Content>
          <div className="space-y-4">
            <Input
              label="Your Name"
              placeholder="Enter your name..."
            />
            <Button variant="primary">Get Started</Button>
          </div>
        </Card.Content>
      </Card>
    </div>
  )
}
```
